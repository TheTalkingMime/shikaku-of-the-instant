Natural Generator

Recursive guillotine cuts with heavy-tailed size distribution. Default generator.

How It Works

The Natural generator uses recursive subdivision — starting with the full grid as one rectangle and recursively splitting it with edge-to-edge cuts. The key innovation is the sigmoid stopping probability: at each step, the algorithm independently decides whether to stop subdividing based on the current rectangle's area.

  1. Start with the full N×N grid as one rectangle
  2. At each rectangle, compute a stopping probability based on its area
  3. If stopping: the rectangle becomes a clue piece
  4. If continuing: choose a cut direction (biased toward the longer dimension) and a random cut position, then recurse on both halves

The Sigmoid Stopping Rule

The probability of stopping at a given rectangle is:

P(stop) = 1 / (1 + (area / M) ^ k)

where M = max(3, size * 0.55)    // characteristic midpoint area
      k = 1.2                    // sigmoid steepness

This creates a smooth transition:

  • At area >> M: probability ≈ 0 — large regions are almost always cut
  • At area = M: probability = 0.5 — coin-flip
  • At area << M: probability ≈ 1 — small regions almost always survive

For a 40×40 board (M ≈ 22):

  Area    P(stop)    Interpretation
  ────    ───────    ──────────────
  1600     0.7%      Full board: almost always cut
   200     8.3%      Still very likely to be cut
    50    25%        Usually cut, sometimes survives
    22    50%        Coin flip
    10    72%        Usually survives
     3    93%        Almost always survives
     1    97%        Tiny — survives

Why Independent Branches Matter

The critical difference from the iterative Guillotine generator is that each branch decides independently whether to stop. One branch might stop at depth 2 (producing a huge 120-area rectangle), while its sibling continues to depth 10 (producing many tiny rectangles). This independent stopping creates the heavy-tailed distribution — a mix of massive blocks and tiny fillers that characterizes professionally generated boards.

Recursive tree showing independent stopping: 1600 / \ 600 1000 STOP / \ (area=600) 450 550 / \ STOP 180 270 (area=550) / \ STOP 80 100 (area=270) STOP / \ 50 50 ... ...

Cut Direction & Position

Cuts prefer the longer dimension (threshold: 1.5× ratio), otherwise random. Cut position is uniformly random — not center-biased. This naturally produces thin strips when cuts land near an edge, adding visual variety.

Key Characteristics

  • Heavy-tailed sizes: areas range from 1 to ~130 on a 40×40 board
  • ~80 rectangles for a 40×40 board (vs ~150 for the iterative Guillotine)
  • Hierarchical structure: large regions subdivided into progressively smaller ones
  • Guillotine property: every cut goes wall-to-wall within its parent rectangle
  • Stochastic count: rect count varies by seed (typically ±15% of the mean)

Pseudocode

function generateNatural(size, seedString):
    rng = mulberry32(hash(seedString))
    M   = max(3, size * 0.55)
    k   = 1.2

    function subdivide(r, c, w, h):
        area = w * h

        // Can't cut a 1-wide strip
        if min(w, h) <= 1:
            emit {r, c, w, h}
            return

        // Sigmoid stopping probability
        stopP = 1 / (1 + (area / M) ^ k)
        if rng() < stopP:
            emit {r, c, w, h}
            return

        // Choose direction (prefer longer axis)
        if   h > w * 1.5: cutH = true
        elif w > h * 1.5: cutH = false
        else:              cutH = rng() < 0.5

        // Uniform random cut position
        if cutH:
            s = 1 + floor(rng() * (h - 1))
            subdivide(r, c, w, s)
            subdivide(r+s, c, w, h-s)
        else:
            s = 1 + floor(rng() * (w - 1))
            subdivide(r, c, s, h)
            subdivide(r, c+s, w-s, h)

    subdivide(0, 0, size, size)
    return finalize(rects)
Guillotine constraint: Because every cut spans the full width or height of its parent rectangle, the resulting partition always has the guillotine-cut property.
