Anchor-Grow Generator

Organic Voronoi-like growth from scattered seed points.

Example Boards (15×15)

How It Works

The Anchor-Grow algorithm creates layouts by scattering seed points across the grid, then growing each one outward into a rectangle until collisions stop further growth.

Phase 1: Seed Placement

Seeds are placed on a jittered grid — a regular grid where each point is offset by a random amount within its cell. This ensures roughly even spacing while avoiding a mechanical lattice pattern.

Jittered grid placement (9 seeds on a 10×10): ┌──────────────────┐ │ · · · │ Each · is a 1×1 seed rect │ │ placed with random jitter │ · · │ within its grid cell. │ │ │ · · │ │ · │ └──────────────────┘

Phase 2: Growth

In repeated rounds, every rectangle tries to expand one cell in a random direction. Growth succeeds only if every cell in the new row/column is unowned. The order is shuffled each round so no rectangle gets a persistent advantage.

Round 1: Each seed tries to grow in a random direction. Round 2: Rectangles grow further, some start colliding. Round N: Growth stalls — all rects are blocked on all sides. ┌───────┬────┬─────┐ │ │ │ │ Shapes are asymmetric — │ A │ B │ C │ each seed grew differently ├──┬────┼────┤ │ depending on what it │ │ │ ├─────┤ collided with. │D │ E │ F │ │ │ │ │ │ G │ ├──┴────┴──┬─┤ │ │ H │I│ │ └──────────┴─┴─────┘

Phase 3: Gap Fill

After growth stalls, any remaining uncovered cells are filled with scan-line packing (the same algorithm used by the Irregular generator), placing rectangles sized near the target average area.

Why Shuffle Growth Order?

If rectangles always grew in the same order, early seeds would dominate and claim most of the space. Shuffling each round ensures fair competition for territory, producing more varied, balanced layouts.

Key Characteristics

  • Organic, asymmetric shapes: no two boards look alike
  • Non-guillotine layouts: rectangle boundaries don't form straight lines across the grid
  • Moderate size variance: seeds near the center grow large, edge/corner seeds stay small
  • Three-phase process: seed, grow, fill — more complex than other generators
Gap-fill artifacts: The scan-line gap filler can sometimes produce small, uniform rectangles in areas that multiple grown rectangles couldn't reach. This is most visible on larger boards where growth collisions leave irregular gaps.

Pseudocode

function generateAnchorGrow(size, seedString):
    rng    = mulberry32(hash(seedString))
    target = round(size ^ 1.4)

    // Phase 1: Scatter seeds on jittered grid
    gridDim = ceil(sqrt(target))
    for i in 0..target:
        gr = floor(i / gridDim)
        gc = i mod gridDim
        r  = clamp(floor(gr * cellH + rng() * cellH))
        c  = clamp(floor(gc * cellW + rng() * cellW))
        place 1×1 rect at (r, c)

    // Phase 2: Grow until no rect can expand
    repeat:
        grew = false
        shuffle(rect order)
        for each rect:
            dirs = shuffle([up, down, left, right])
            for each dir:
                if canGrow(rect, dir):
                    expand(rect, dir)
                    grew = true
                    break
    until !grew

    // Phase 3: Fill gaps with scan-line packing
    for each uncovered cell (r, c):
        place best-fit rectangle near avgArea

    return finalize(rects)
