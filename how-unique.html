Unique Generator

Constraint-based placement with solution uniqueness verification.

Example Boards (10×10)

Shown at 10×10 because uniqueness verification is most effective at smaller sizes.

How It Works

The Unique generator is the only algorithm that attempts to produce puzzles with exactly one valid solution. It combines two techniques:

  1. Smart clue placement — choosing the most-constrained position for each number
  2. Backtracking solver — verifying the puzzle has a unique solution

Step 1: Generate a Base Tiling

Uses the Irregular scan-line packer to create a tiling. This serves as the intended solution.

Step 2: Smart Number Placement

Instead of placing each clue number at a random cell within its rectangle, the algorithm picks the cell that minimizes the number of alternative rectangles that could contain it.

Example: a 3×2 rectangle (area = 6) For each cell, count how many distinct rectangles of area 6 could contain that cell: Cell (2,4): 8 alternatives Cell (2,5): 6 alternatives Cell (2,6): 4 alternatives Cell (3,4): 7 alternatives Cell (3,5): 5 alternatives Cell (3,6): 3 alternatives ← pick this one! Corner cells near grid edges have the fewest alternatives → most constrained → better puzzles.

Step 3: Uniqueness Verification

For boards up to 20×20, a backtracking Shikaku solver counts solutions. The solver uses two optimizations:

  • Most-constrained first: clues with fewer possible placements are processed first, pruning the search tree early
  • Forward checking: after placing each rectangle, verify that every remaining clue still has at least one valid option

If the puzzle has multiple solutions, the generator retries with a modified seed (up to 20 attempts for small boards).

Operation Limits

To keep generation fast, the solver uses operation limits (not time limits, so seeds always reproduce the same board):

  Board size    Op limit     Max attempts
  ──────────    ────────     ────────────
  5×5–7×7       100,000            20
  8×8–10×10     300,000            20
  11×11–15×15   800,000            20
  16×16–20×20  2,000,000            8
  21×21+       (skipped)             1
Large boards: For boards larger than 20×20, uniqueness checking is skipped entirely (the search space is too large). Smart clue placement is still used, which produces significantly more constrained puzzles than random placement — just without a uniqueness guarantee.

Key Characteristics

  • Unique solutions (verified for boards ≤ 20×20)
  • Highly constrained clue positions: numbers placed at corners and edges
  • Irregular tiling base: uses scan-line packing for the rectangle layout
  • Slower generation: solver verification adds computation time
  • Retry mechanism: multiple seed attempts to find unique puzzles

Pseudocode

function generateUnique(size, seedString):
    for attempt in 0..maxAttempts:
        rng   = mulberry32(hash(seedString + attempt))
        rects = irregularTile(size, rng)

        // Smart placement: minimize alternatives
        for each rect:
            for each cell (r, c) in rect:
                count = countPossibleRects(r, c, rect.area, size)
                if count < bestCount:
                    bestCell = (r, c)
            rect.numberPos = bestCell

        // Verify uniqueness (small boards only)
        if size <= 20:
            solutions = countSolutions(size, clues, opLimit)
            if solutions == 1:
                return puzzle  // found unique!
            continue       // try next seed

        return puzzle    // large boards: return with smart placement

function countSolutions(size, clues, maxOps):
    // Enumerate placements, sort by constraint
    order = sortBy(clues, fewestOptionsFirst)

    function backtrack(depth):
        if solutions >= 2 or ops >= maxOps: return
        if depth == clues.length: solutions++; return

        for each placement of order[depth]:
            if overlaps: continue
            place on grid
            if allRemainingFeasible():
                backtrack(depth + 1)
            remove from grid
