Irregular Generator

Scan-line packing with interlocking, non-guillotine layouts.

Example Boards (15×15)

How It Works

Instead of splitting, the Irregular generator fills the grid by scanning left-to-right, top-to-bottom and placing rectangles one at a time into the remaining empty space.

  1. Scan for the next uncovered cell (reading order)
  2. Measure available space: for each possible width, find the maximum height before hitting a covered cell
  3. Generate candidates: try (width, height) pairs near the target average area (N² / clueCount), with a range of ±2 around the ideal height
  4. Weighted random pick: candidates closer to the average area are more likely, using weight 1 / (1 + |area - avgArea|)
  5. Place the rectangle and mark cells as covered
  6. Repeat until the entire grid is filled
Scan-line packing — rectangles interlock: ┌───────┬─────┬──────┐ │ │ │ │ Rectangles fit into gaps left │ 3×4 │ 2×3 │ │ by previous placements. │ │ │ 5×2 │ │ ├─────┤ │ No guillotine line exists ├──┬────┤ │──────┤ that bisects the whole grid. │ │ │ │ │ │ │ │ │ │ └──┴────┴─────┴──────┘

Why Weighted Selection?

The weighting toward the average area prevents the algorithm from placing all tiny or all huge rectangles. The formula 1/(1+|area - avgArea|) gives highest weight to exact average matches, with diminishing weight for outliers — but outliers can still be picked, creating a natural mix of sizes.

Key Characteristics

  • Interlocking layouts: no edge-to-edge guillotine lines through the whole grid
  • Relatively uniform sizes: most rectangles cluster near the average area
  • Scan-order bias: top-left rectangles have more placement freedom than bottom-right ones
  • Guaranteed coverage: every cell is always filled (scan-line ensures no gaps)
Scan-order bias: Because the algorithm scans left-to-right, top-to-bottom, rectangles placed early (top-left) have more freedom in dimensions, while later placements (bottom-right) must fit into remaining gaps. This can produce slightly different visual "feels" between regions of the board.

Pseudocode

function generateIrregular(size, seedString):
    rng     = mulberry32(hash(seedString))
    target  = round(size ^ 1.4)
    avgArea = size * size / target
    covered = size × size grid of false
    rects   = []

    for each cell (r, c) in reading order:
        if covered[r][c]: continue

        // Measure available space
        maxW = consecutive uncovered cells to the right
        for w in 1..maxW:
            maxH[w] = max rows of width w from (r,c)

        // Build candidates near avgArea
        candidates = []
        for w in 1..maxW:
            idealH = clamp(round(avgArea / w), 1, maxH[w])
            for h in (idealH-2)..(idealH+2):
                candidates.add(w, h)

        // Weighted pick (prefer candidates near avgArea)
        pick = weightedRandom(candidates,
                              weight = 1 / (1 + |area - avgArea|))

        rects.add(pick)
        mark cells as covered

    return finalize(rects)
