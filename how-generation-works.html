How Board Generation Works

A guide to the five Shikaku puzzle generators and how they create boards.

Overview

Every generator produces a complete tiling of the grid — a set of non-overlapping rectangles where every cell belongs to exactly one rectangle. Each rectangle becomes a clue, with its area as the number shown to the player.

All generators share two common pieces:

  1. Seeded PRNG — a deterministic Mulberry32 random number generator, so the same seed always produces the exact same board
  2. Number placement — each clue number is positioned at a random cell within its rectangle (or at the most-constrained cell, in the Unique generator)

The Generators

Default

Natural

Recursive guillotine cuts with a sigmoid stopping rule. Produces heavy-tailed size distributions — large blocks coexist with tiny fillers. Mimics professionally generated boards.

Guillotine

Iterative area-weighted edge-to-edge splits. Fast, clean, structured layouts with relatively uniform rectangle sizes.

Irregular

Scan-line packing that fills the grid left-to-right, top-to-bottom. Produces interlocking, non-guillotine layouts with varied shapes.

Anchor-Grow

Organic Voronoi-like growth from scattered seed points. Rectangles expand outward until they collide, producing asymmetric, natural-looking layouts.

 Advanced

Unique

Constraint-based placement with a backtracking solver to verify unique solutions. Smart clue positioning maximizes puzzle constraint.

Seed Format

{gridSize}_{6-digit-number}

Examples:
  5_003921    →  5×5 board, seed 003921
  20_847291   → 20×20 board, seed 847291
  40_100042   → 40×40 board, seed 100042

The grid size is encoded in the seed so that when sharing, the game automatically selects the correct board size.

Common Limitations

No uniqueness guarantee (most generators): Only the Unique generator verifies that a puzzle has exactly one solution. The other four generators may produce puzzles with multiple valid partitions.
No difficulty tuning beyond size: Two boards of the same size can have very different difficulty. The generators don't analyze or control for this — difficulty comes primarily from grid size and rectangle distribution.
